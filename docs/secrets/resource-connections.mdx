# Resource Connections

Resources in a project can reference project-level secrets and variables directly from their configuration fields. Instead of hardcoding a value, you insert a reference expression that the platform resolves to the correct value for the active environment at runtime.

Define a secret or variable once at the project level, set its value per environment, and every resource referencing it automatically receives the right value for that environment.

## How reference expressions work

- Secret reference pattern: `${blueprint.self.secrets.SECRET_NAME}`
- Variable reference pattern: `${blueprint.self.variables.VARIABLE_NAME}`

At runtime, the platform resolves each expression to the value for the active environment:

- **Variables:** if an environment-specific override exists, the platform uses it. If no override exists, the platform falls back to the project-level default.
- **Secrets:** a value must be set at the environment level before deployment. Secrets have no project-level default. If no environment value is set, the resource receives no value for that field.

## Using secret references in resource configuration

:::info Interactive Demo
*An interactive walkthrough for this flow will be added here.*
:::

Resource configuration forms include **SecretRefWidget** autocomplete fields for secrets. These fields surface all secrets defined in the project.

1. Open the resource configuration.
2. Locate a field that accepts a secret reference.
3. Type the secret name or scroll the list to find it.
4. Select the secret. The field populates with the expression `${blueprint.self.secrets.SECRET_NAME}` automatically.
5. Save the resource configuration.

> **Note:** If the secret has no value set for the active environment, the resource will not receive a value for that field. Set the secret value per environment before deploying.

## Using variable references in resource configuration

:::info Interactive Demo
*An interactive walkthrough for this flow will be added here.*
:::

Resource configuration forms include **VariableRefWidget** autocomplete fields for variables. These fields surface all variables defined in the project.

1. Open the resource configuration.
2. Locate a field that accepts a variable reference.
3. Type the variable name or scroll the list to find it.
4. Select the variable. The field populates with the expression `${blueprint.self.variables.VARIABLE_NAME}` automatically.
5. Save the resource configuration.

> **Note:** If no environment-specific override exists for a variable, the platform uses the project-level default at runtime.

## Auto-Inject variables

Variables with **Auto-Inject** enabled are injected into all resources at runtime without any explicit reference in any resource configuration field. You do not need to add a reference expression for Auto-Inject variables — they are already present at runtime.

Auto-Inject is not available for Secrets.

To configure the Auto-Inject flag on a variable, see [Project Level Secrets and Variables](./project-level-secrets.mdx).

## Copying a reference expression manually

Use this approach when a configuration field does not have an autocomplete widget but accepts the reference expression syntax.

:::info Interactive Demo
*An interactive walkthrough for this flow will be added here.*
:::

1. Navigate to the **Secrets & Variables** page.
2. Find the secret or variable row.
3. Click **Copy $ Reference**.
4. The reference expression is copied to your clipboard.
5. Paste it into any resource configuration field that accepts free text.
